Sample Projects

I emphasize analysing the requirements fully first, then creating a design. For more advanced students, a UML design model. The design model can be used to generate skeleton classes to begin writing the code. I encourage a test-driven development approach, when a student is at an appropriate level to understand it fully and benefit from it.

I have compiled a collection of good example solutions to projects and common problems to provide guidance of how to approach design and development.

Grocery Warehouse Management System

Objective

Write a Graphical User Interface (GUI) or command-line application to manage customers, sales and inventory for a Grocery Warehouse.

Topics to know in order to complete the project

  1. Java syntax and programming basics
  2. How to implement Classes with instance variables and methods
  3. How to implement a design using OO Concepts such as: Polymorphism, Encapsulation, and Composition
  4. Abstract classes
  5. Interfaces
  6. Error Handling
  7. GUI Design: Swing Components and Layout Managers
  8. Event-driven programming (ActionListeners)
  9. Collections
  10. File I/O

Difficulty

Suitable for advanced students only. This could be 20-40 hours of work depending on how well the topics are understood. The assignment outlined is a good example of a difficult specification that has: verbose explainations, restating requirements in slightly different ways or adding extra detail in different sections, some prescriptive demands e.g. define a Customer class with addSale method, and some conflicting requirements that need some compromise to accomodate.
